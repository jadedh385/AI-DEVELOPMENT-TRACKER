# Stage 1.4 — Item Enrichment: Preference Gate Decisions

**Date:** 2026-07-06
**Stage:** 1.4 — Item Enrichment (one-line "why it matters" summaries)

## Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Summarization trigger | **During ingest** | Feed is always ready to read; cost is predictable per run |
| Model | **Claude Haiku 4.5** (`claude-haiku-4-5`) | $1/$5 per MTok — ~$0.45/month at personal scale; sufficient for short summaries |
| LLM failure fallback | **Leave `summary` null** | Simple, honest; item still usable (title + link); avoids stale/wrong text |
| Caching | **Skip items where `summary IS NOT NULL`** | Idempotent; only new items cost tokens; re-running ingest never double-bills |

## Notes

- Requires `ANTHROPIC_API_KEY` in `.env` (Anthropic API account with credit loaded)
- Estimated cost: ~$0.003 per 10 items ≈ $0.015/day at 50 new items/day ≈ $0.45/month
- `summary String?` already exists in Prisma schema (no migration needed)
- Prompt caching on the system prompt block will reduce per-call cost further
